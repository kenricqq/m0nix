# Connecting the Dots

1. If learning is a pipeline:

- Solon.md serves as inbound filter
  - process, comprehend
- Athena.sh expands on ideas and theories
  - try stuff out, run experiments

2. If learning is questions

- Solon.md provides meaning. Why explore <X>?
- Athena.sh provides the "what". What about <X> is so special?

## solon.md

1. aggregate/filter/summarize: YouTube, RSS blogs, X.com feeds...
2. encourage active learning; comprehension quiz, think prompting by asking relevant questions (e.g. what changed about your understanding of the world), theories to test
3. feynman technique: software engineers has had rubber duckies for a long time

## athena.sh

1. experimentation sandbox (engineering / writing)
2. feedback tracking, update systems accordingly
3. instruction layer (run research) on rss

- e.g. for every new blog on https://svelte.dev/blog/rss.xml, explore each community showcase project, get the github stats (stars, commits, clarity of docs, purpose, similar popular projects etc)
